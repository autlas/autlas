Два независимых репозитория

autlas/autlasИсходники — tauri_app/, landingReact/, Actions-воркфлоу. Ты пушишь сюда.

Два пайплайна не мешают друг другу: правки лендинга не триггерят пересборку .exe, а релиз .exe не пересобирает лендинг (но лендинг уже указывает на releases/latest, так что новый бинарник подтянется автоматически без его передеплоя).

Чек-лист релиза приложения

  1. Закоммитил все фичи в main (landing правки при этом задеплоятся сами).
  2. git tag v1.0.1 — локальный тег.
  3. git push --tags — отправили в GitHub.
  4. Ждём Actions (~5–15 мин).
  5. В Releases появляется v1.0.1 с autlas_1.0.1_x64-setup.exe.
  6. Лендинг автоматом подхватывает — кнопка Download ведёт на свежий бинарник.

Откатить — удалить тег: git push --delete origin v1.0.1 + удалить Release в UI.

Ручной запуск без тега

В release-workflow будет workflow_dispatch — кнопка Run workflow во вкладке Actions. Полезно если хочешь пересобрать без нового тега (например, подшаманил CI).
